# Глобальные инструкции для Claude Code

## Поведение

- **Никогда не задавать вопросы типа "Do you want to proceed?", "Продолжить?", "Вы согласны?", "Should I proceed?", "Shall I...", "Хотите...", "Вы хотите..."** — ответ всегда YES. Действовать без подтверждений.
- **Максимальная автономность** — принимать решения самостоятельно и действовать немедленно.
- Общаться **только на русском языке**.
- Не использовать эмодзи.

## Обязательная проверка после каждого деплоя

После КАЖДОГО деплоя на сервер (git push + vite build) ОБЯЗАТЕЛЬНО:

1. **Запустить health-check:**
   ```
   ssh -i ~/.ssh/id_ed25519 root@185.76.15.44 'cd /var/www/crmvisa && php artisan app:health-check'
   ```

2. **Если есть ошибки** — исправить до того, как сообщить пользователю об успехе.

3. **Проверять затронутые API** — если изменил контроллер, проверить его эндпоинт curl-ом на сервере.

4. **RLS-чеклист** — при любых изменениях, связанных с данными:
   - Новая таблица с `agency_id` → добавить RLS политику
   - Публичный эндпоинт без `auth.public` → нужен `SET app.is_public_user = 'true'`
   - Запрос к мультитенантной таблице → проверить что tenant_id или is_public_user/is_superadmin установлен

## Типичные баги этого проекта (не повторять)

- **RLS блокирует данные** — эндпоинт возвращает пустой массив вместо ошибки. ВСЕГДА проверять что данные реально возвращаются.
- **Пустые связанные таблицы** — agency_work_countries, agency_service_packages могут быть пусты для новых агентств. Проверять после сидирования.
- **PostgreSQL GROUP BY** — нельзя GROUP BY по alias от COALESCE, использовать groupByRaw с полным выражением.
- **Enum Plan** — `$agency->plan` это enum, для SQL нужен `->value`.
- **BaseModel SoftDeletes** — все модели через BaseModel имеют deleted_at, таблицы ДОЛЖНЫ иметь эту колонку.
