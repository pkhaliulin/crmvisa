# Чеклист качества разработки — ВСЕГДА проверять

## Уроки из реальных багов

### Баг 1: Голые i18n ключи (common.edit, common.delete)
**Причина:** Использовали `$t('common.edit')` в шаблоне, но ключ не был добавлен в ru.json/uz.json.
**Правило:** При использовании ЛЮБОГО i18n ключа — СРАЗУ проверить что он существует в ОБОИХ файлах (ru.json и uz.json). Не откладывать.

### Баг 2: Удаление без подтверждения
**Причина:** Кнопка "Удалить" вызывала API напрямую, без модала подтверждения. Пользователь случайно удалил "married".
**Правило:** ЛЮБОЕ удаление — только через модал подтверждения. Паттерн: `deleteTarget = item` -> модал -> подтверждение -> API.

### Баг 3: Хардкод строк на русском
**Причина:** Кнопки/метки написаны напрямую на русском, не через $t(). При переключении на UZ — текст остается русским.
**Правило:** НИ ОДНА видимая пользователю строка не должна быть хардкодом. Только через $t().
